Attal is developped under Linux environment.
Sources are written in C++ (using Qt 2.2 libraries).
Note : Qt is released under GPL and you can download it from : ftp://ftp.trolltech.com/qt/source/
If you want to know the development steps, look at our roadmap

You can also browse the CVS on Attal's project site (hosted by Sourceforge) in order to be up-to-date with the evolutions of Attal.

Shortview of the sources.

We will trying here to give a quick shortview of sources in order to help new developers to join us.

There is a few folders in the source :
- client : code specific for the client (GUI, ...).
- server : code specific for the server (GUI, game engine, ...).
- common : code in common for the client and the server (network, game structures like map, lords...).
- ia : code managing the ai, both for the fight phase (this code is also used by the server when someone fights against creatures which are not owned by any player) and for the adventure phase. - editor : code specific for the map editor.
- themes : only data files (and pics) for the theme management.

Theme management

The theme management is one of the most important feature of Attal : it will allow to create totally different games (different graphics, different rules...). Information and data which are theme-specific have to be in the 'themes' directory, where each sub-directory defines a new theme. The name of the sub-directory will be the name of the theme (for example 'cyberpunk'). In these sub-directories are other folders :
'data', where rules are defined and 'images' where are graphics. There could be more folders (for example for the sound...) Theses folders ('data', 'images', ...) could be divided with other sub-folders
