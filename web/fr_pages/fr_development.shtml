Attal est développé sous Linux.
La programmation est effectuée en C++ en utilisant la bibliothèque QT à partir de la version 2.2

Cette bibliothèque est distribuée sous GPL et peut être téléchargée à l'adresse suivante :
ftp://ftp.trolltech.com/qt/source/

Pour suivre les différentes étapes du développement du jeu, n'hésitez pas à consulter notre planning de développement.

Vous pouvez de même consulter notre cvs sur le site du projet sur Sourceforge pour suivre directement les évolutions du code.


Présentation de l'architecture.

Cette partie à pour but de donner un aperçu de la structure du source afin que de nouveaux programmeurs puissent plus facilement nous rejoindre (et aussi pour rafraîchir les idées des autres).
Le code source est décomposé en répertoires :

  • client : fichiers propres au client, en particulier l'interface graphique.
  • server : fichiers propres au serveur, en particulier l'interface et le moteur de jeu.
  • common : fichiers communs au serveur et au client, ce qui comprend une partie de la gestion réseau, mais surtout les structures du jeu (gestion de carte, de lords, etc.)
  • editor : fichiers propres à l'éditeur de carte.
  • themes : uniquement des fichiers de données utilisés dans la gestion des thèmes.


La gestion des thèmes.

L'utilisation de thèmes permettant de transformer du tout au tout le jeu, tant son apparence que les règles le régissant est un élément primordial du développement d'Attal. A cette fin, les données définissant les thèmes doivent être placées dans le répertoire 'themes' qui est organisé en sous-répertoire décrivant chacun un thème. Le nom du sous-répertoire désigne le nom du thème (par exemple 'cyberpunk'). Chacun de ces répertoires est composé d'autres sous-répertoires :
'data', qui contiendra les règles à utiliser et 'images' qui contient les graphiques. Ces répertoires peuvent eux-même être décomposés en sous-répertoires.
